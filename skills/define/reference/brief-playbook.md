# Brief playbook — probe bank for `BRF-001`

The anchor. Spend the most time here. Working Backwards: the press release and the
customer quote come first; if you can't write the one-line customer benefit, keep
digging. Shared posture + quality bar: `../product-model/discovery.md`.

## Dig each until concrete, specific, confirmed
- **Problem** — whose pain exactly, how acute, how often, what they do **today instead**
  (current alternatives), and the **cost of doing nothing**. A problem nobody works
  around isn't a problem.
- **Why now** — what changed (tech, market, regulation, behaviour) that makes this the
  moment.
- **Target user** — the *one* segment to win first. Reject "everyone".
- **Value in one line** — the press-release headline (customer benefit, not feature).
- **North Star** — the single metric that proves value is delivered. Push back on vanity
  metrics (signups, page views) — prefer the metric that only moves when the user wins.
- **Non-goals** — what the product deliberately won't do (the scope it refuses).
- **Top risk / unknown** — the assumption that, if wrong, sinks the product.

## Brownfield additions
- `Vision implicite` — what the code optimizes **today** (from `audit-codebase`).
- `Vision cible` — where the user wants to take it. **The delta is the roadmap.**

## Quality bar (stop a round only when…)
The answer is concrete, specific, and confirmed. Red flags = keep digging: "everyone" as
a user, a North Star you can't measure, a vision that just restates the feature list, an
empty FAQ answer. Restate the press release + FAQ, get an explicit yes, *then* write.

## Anti-patterns
- ❌ Asking only "project name?" + "main features?".
- ❌ Writing `BRF-001` before problem + vision + North Star are confirmed.
- ❌ Treating the template as a top-to-bottom questionnaire.
